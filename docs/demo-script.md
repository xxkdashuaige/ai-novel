# Demo 视频演讲稿

## 1. 开场介绍

大家好，这个项目是一个 AI 小说转剧本工具，使用 Spring Boot、Spring AI、Vue 3 和 Element Plus 实现。

它的目标是把 3 个章节以上的小说文本自动转换成结构化剧本 YAML，让作者快速得到一份可以继续编辑、校验和打磨的剧本初稿。

## 2. 技术栈说明

后端使用 Spring Boot 提供 REST API，使用 Spring AI 通过 OpenAI-compatible 客户端接入 DeepSeek 模型。前端使用 Vue 3 和 Element Plus 构建交互界面。剧本结构通过 Java DTO 管理，再转换为 YAML，避免直接信任 AI 输出的纯文本。

## 3. 启动与 AI 状态

项目分为前后端两个服务。后端默认运行在 8080，前端默认运行在 5173。

如果没有配置 API Key，项目会使用规则解析器兜底，仍然可以完成基础演示。如果配置了 DeepSeek Key，后端会启用 Spring AI 调用模型。

页面顶部会展示当前 AI 状态，包括模型、Key 是否配置、当前是 AI 模式还是规则解析器模式。

## 4. 一键转换小说

接下来我粘贴一段 3 章以上的小说文本，然后点击“一键转换”。

转换过程中，页面会显示阶段提示，例如准备文本、调用 AI 或规则解析、结构化剧本、校验质量、保存历史。这样用户不会不知道系统是否正在工作。

转换完成后，左侧可以看到原始小说，右侧可以看到生成的 YAML 剧本。

## 5. YAML Schema 与结构化剧本

这个项目不是简单生成一段文本，而是生成结构化 YAML。

剧本 Schema 采用“作品 -> 章节 -> 场景 -> 节拍”的层级。每个场景包含地点、时间、人物和 beats。beat 可以是对白、动作、旁白或转场。

这样设计的原因是：章节保留小说原结构，场景适合后续拍摄和分镜，beat 是最小可编辑单元，方便后续做 AI 编辑和导出。

## 6. 校验与质量评分

生成后，系统会自动校验 YAML 结构。

右侧会展示校验结果和剧本质量评分。质量评分会检查章节数量、场景是否完整、人物是否一致、地点和时间是否缺失等问题。

如果质量分较低，系统会展示具体问题和优化建议。

## 7. 一键修复一致性

如果某个角色在对白中出现了，但没有被列入当前场景的 characters 列表，系统会给出一致性提醒。

点击“一键修复一致性”后，后端会扫描每个场景的对白 speaker，并自动补全到场景 characters 中，然后重新生成 YAML 和校验结果。

这个功能解决的是结构一致性问题，不会随意改写剧情内容。

## 8. 剧本可视化预览

转换完成后，可以在下方查看剧本可视化预览。

章节预览会按章节和场景展示对白、动作、旁白和转场，比直接看 YAML 更适合作者阅读。

## 9. 角色关系图和场景分镜表

角色关系图会根据同一场景中共同出现的角色自动连线。节点数字表示角色出场场景数，连线粗细表示同场次数。

场景分镜表会把每条对白、动作或旁白整理为表格，包含章节、场景、地点、时间、人物、类型和内容。这个表可以导出 CSV，方便继续整理拍摄计划。

## 10. AI 辅助编辑

这个项目的 AI 辅助不只是生成文本，还支持局部编辑。

在单条 beat 上，可以选择优化台词、压缩对白、补充动作、补充镜头语言、增强冲突或改成短剧风格。

在场景级别，可以选择优化整场对白、补充动作细节或增强冲突节奏。应用后，系统会重新生成 YAML、校验结果和质量评分。

## 11. 历史记录与导出

每次转换都会保存到后端历史记录中，默认保留最近 20 条。历史记录支持搜索、按生成模式筛选、加载和删除。

导出方面，项目支持 YAML、JSON、TXT、分镜 CSV、Word-compatible DOC，也可以通过浏览器打印保存 PDF。

## 12. 总结

总结一下，这个项目完整展示了 Spring Boot + Spring AI + Vue 3 的前后端整合能力。

AI 辅助体现在小说转换、剧本结构化、台词优化、动作补充、场景冲突增强和质量分析等多个环节。即使没有 API Key，规则兜底也能保证项目可以稳定演示。

后续如果继续扩展，可以加入数据库、多用户管理、专业 DOCX/PDF 导出和更完整的拍摄分镜字段。
